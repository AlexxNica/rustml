A matrix with elements of type T.

An Option is returned which is None if the matrices cannot be multiplied because the number of columns of the left matrix is not equal to the number of rows of the right matrix.

Because the trait Mul is implemented for matrices where T is f32 or f64 such matrices can be multplied with the * operator.

Multiplication of matrices

To multiply two matrices the underlying BLAS implementation is used. By default this is CBLAS. In many cases performance can be greatly improved when switching to ATLAS. For a detailed description on how to optimize the numeric computations please read the separate documentation on this topic available here.

Example

use rustml::*;

let a = mat![
    1.0f32, 5.0, 2.0; 
    2.0, 2.0, 3.0 
];
let b = mat![
    3.0, 7.0, 4.0, 8.0;
    4.0, 2.0, 1.0, 4.0;
    5.0, 2.0, 1.0, 9.0
];
let c = (a * b).unwrap();
assert_eq!(c.row(0).unwrap(), &[33.0, 21.0, 11.0, 46.0]);
assert_eq!(c.row(1).unwrap(), &[29.0, 24.0, 13.0, 51.0]);

assert_eq!(m.cols(), 2);
// all values are in the interval [0, 1)
assert!(m.values().all(|&x| x < 1.0 && x >= 0.0));
println!("{}", m);

fn set(&mut self, row: usize, col: usize, newval: T) -> bool

Replaces the element at row row (indexing starts at zero) and column col with the new value newval. Returns true on success and false on failure, i.e. if the row or column does not exist.

use rustml::*;

let mut m = mat![
    1.0, 1.5; 
    2.0, 2.5;
    5.0, 5.5
];
assert_eq!(m.get(1, 0).unwrap(), &2.0);
m.set(1, 0, 8.0);
assert_eq!(m.get(1, 0).unwrap(), &8.0);
assert_eq!(m.set(3, 0, 8.9), false);
